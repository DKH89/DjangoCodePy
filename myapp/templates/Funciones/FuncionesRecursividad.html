11. Recursividad en Funciones

La recursividad en Python es un concepto poderoso que implica que una función se llame a sí misma directa o indirectamente para resolver un problema. Es una técnica de programación que permite dividir un problema en subproblemas más pequeños y resolverlos de manera recursiva hasta alcanzar una solución.Cuando una función se llama a sí misma dentro de su propio cuerpo, se produce lo que se conoce como llamada recursiva. Esto puede ser útil para resolver problemas que pueden ser definidos en términos de sí mismos, como el cálculo de factoriales, la generación de secuencias numéricas, la exploración de estructuras de datos recursivas como árboles o grafos, entre otros.






11.1 Enfoque de Recursividad

El enfoque de la recursividad en Python es fundamental para resolver problemas complejos dividiéndolos en casos más simples y manejables. Este enfoque se basa en dos elementos principales: el caso base y el caso recursivo. El caso base proporciona un punto de salida para la recursión, definiendo un resultado conocido que evita llamadas infinitas y posibles desbordamientos de pila. Por otro lado, el caso recursivo es el paso en el que la función se llama a sí misma con argumentos modificados, lo que permite que el problema se reduzca progresivamente hasta alcanzar el caso base. Este enfoque proporciona una manera elegante y concisa de abordar problemas complejos, pero es importante utilizarlo con cuidado para evitar bucles infinitos y considerar la eficiencia del algoritmo en cada situación específica.

11.3 Caracteristicas de una función recursiva

Las funciones recursivas tienen algunas características distintivas que las diferencian de las funciones convencionales.

● Llamadas a sí misma: Una función recursiva se llama a sí misma dentro de su propio cuerpo para resolver un problema de manera repetida. Esta llamada recursiva es esencial para la naturaleza recursiva de la función.

● Caso base: Toda función recursiva debe tener al menos un caso base. Este caso base proporciona una condición de salida para la recursión y evita que la función se llame a sí misma indefinidamente. Es el punto en el que se resuelve el problema sin necesidad de más llamadas recursivas.

● Problema reducido: En cada llamada recursiva, el problema se reduce o se acerca al caso base. Esto asegura que la recursión eventualmente llegue al caso base y termine, evitando así una recursión infinita.

● Eficiencia: Aunque la recursión puede ser una forma elegante de resolver ciertos problemas, puede ser menos eficiente en términos de tiempo y memoria en comparación con enfoques iterativos. Esto se debe a que cada llamada recursiva requiere espacio en la pila de llamadas, lo que puede ser costoso en términos de memoria.

● Legibilidad y simplicidad: En algunos casos, la recursión puede hacer que el código sea más legible y conciso al abstraer la lógica repetitiva en una sola función. Esto puede hacer que el código sea más fácil de entender y mantener, especialmente para problemas que son naturalmente recursivos.
