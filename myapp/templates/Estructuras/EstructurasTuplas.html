8.Tuplas

En Python, una tupla es una estructura de datos que comparte similitudes con las listas, pero con una característica fundamental: son inmutables. Esto significa que una vez que se crea una tupla, no se pueden modificar sus elementos. Las tuplas se utilizan para almacenar una colección ordenada de elementos de diversos tipos, al igual que las listas, pero se prefieren en situaciones donde se requiere asegurar que los datos no cambien después de su creación. Esta característica las hace útiles en contextos donde la integridad de los datos es crucial, como en la definición de constantes o en la representación de datos que no deben ser alterados, como las coordenadas geográficas.






8.1 Creación de Tuplas

Las tuplas en Python se crean utilizando paréntesis () y separando los elementos dentro de la tupla con comas. Esta sintaxis es simple y directa, lo que facilita la creación de tuplas de forma rápida y eficiente. Por ejemplo, una tupla que contenga los días de la semana se puede definir de la siguiente manera: (lunes, martes, miércoles, jueves, viernes, sábado, domingo). Cada elemento de la tupla está separado por comas dentro de los paréntesis, lo que define claramente los límites de la tupla y los elementos que contiene.

8.2 Acceso a los elementos de una tupla

Los elementos de una tupla en Python se acceden utilizando índices, al igual que en las listas. Esto significa que cada elemento en una tupla tiene una posición específica, comenzando desde el índice 0 para el primer elemento, 1 para el segundo elemento, y así sucesivamente. Para acceder a un elemento específico de una tupla, simplemente se utiliza su índice dentro de corchetes después del nombre de la tupla. Por ejemplo, si tenemos una tupla llamada mi_tupla que contiene los elementos (1, 2, 3, 4, 5), podemos acceder al tercer elemento (que es 3) utilizando mi_tupla[2], ya que los índices en Python comienzan desde 0.

8.3 Métodos de las Tuplas

Las tuplas en Python se distinguen por ser estructuras de datos inmutables, lo que significa que una vez creadas, no pueden modificarse. Esta característica limita el conjunto de métodos disponibles en comparación con las listas, que son mutables. Sin embargo, las tuplas aún proporcionan algunos métodos útiles para operaciones como contar ocurrencias de elementos y encontrar el índice de un valor específico dentro de la tupla. Aunque son menos versátiles en términos de manipulación directa, las tuplas son valiosas por su estabilidad y garantía de que los datos no cambiarán después de su creación, lo que las hace ideales para situaciones donde la integridad de los datos es primordial.
